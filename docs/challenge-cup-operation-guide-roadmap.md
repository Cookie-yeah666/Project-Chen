# AI 桌宠分步操作指引路线图

更新时间：2026-07-16（Asia/Shanghai）

## 产品定位

通俗名称：AI 桌宠分步操作指引助手

学术名称：基于屏幕全域感知、以虚拟桌宠为交互载体的交互式分步操作引导系统

一句话卖点：它不是给用户一篇教程，而是在用户当前屏幕上直接让桌宠指出下一步该点哪里。

## V1.0 本届挑战杯落地版本

目标：依托屏幕实时感知 + 互联网教程自动解析，以虚拟 AI 桌宠作为可视化指引载体，通过角色位移、指向动画和气泡文字，实现零基础用户的软件部署分步引导。

核心边界：

- 不做自动点击，用户始终自己操作。
- 不画红圈、矩形或额外透明层箭头；所有可视化指引由桌宠移动、八方向指向动画和气泡完成。
- 每次只给一个目标动作，避免一次性输出长教程。
- 用户可以在每一步选择“我完成了”“重新识别”“退出”。
- 页面滚动、跳转、弹窗或加载变化后，系统先重新识别当前步骤，不擅自跳到下一步。
- 高精度定位优先，8 秒内无法稳定定位时回退到普通定位方式。

V1.0 数据流：

1. 用户在 F11 设置中启用“分步指引”，填写独立 API Key、API 地址、模型和目标软件。
2. 系统根据目标软件联网检索官方文档、技术博客和社区教程。
3. AI 将教程清洗成结构化步骤队列。
4. 系统截图并识别当前屏幕上的按钮、链接、输入框、滚动条、弹窗等控件。
5. 指引中台融合“教程步骤”和“当前屏幕”，确定当前唯一目标控件。
6. 桌宠移动到目标附近，播放指向动画，气泡给出短提示。
7. 用户完成后点击“我完成了”，或页面变化后点击“重新识别”。
8. 循环执行，直到安装/配置流程结束。

V1.0 优先演示任务：

- 下载软件：OpenClaw、Steam、Codex、Claude 客户端。
- 注册/登录/安装流程中的按钮定位。
- 页面里找按钮、找入口、找设置项。

## V1.1 丝滑滚动与重新识别版本

目标：优化网页滚动、页面跳转、弹窗加载后的指向稳定性。

已落地策略：

- 屏幕指纹变化后不再自动进入下一步。
- 先等待页面短暂稳定，再重新定位当前步骤目标。
- 如果页面仍在变化，继续短等待，不让桌宠频繁跳动。
- 如果当前目标暂时找不到，保留当前步骤，提示用户把页面停在目标附近；滚动类步骤提示用户继续滚动，看到后自动指向。
- 增加“重新识别”按钮，用户可以主动让桌宠重新看当前屏幕。

## V1.2 屏幕进度感知版本

目标：让桌宠不仅知道“屏幕变了”，还要初步判断“用户是不是已经完成了当前步骤”。

已落地策略：

- 页面变化后先等待短暂稳定，再用轻量 Vision JSON 判断当前步骤是否完成。
- 如果高置信度看到下一步目标或完成后的界面，自动进入下一步并移动桌宠指向新目标。
- 如果只是打开了搜索框、弹出了输入框、页面还停在当前步骤，继续重新定位当前目标。
- 屏幕变化轮询和稳定等待缩短，提高打开搜索框、页面跳转后的响应速度。

## V1.2.1 指引按钮交互优化

目标：让“我完成了 / 重新识别 / 退出”三个按钮更容易点、更快响应。

已落地策略：

- 指引面板纳入桌宠窗口交互热区，避免透明窗口鼠标穿透导致按钮点不准。
- 按钮改为 pointerdown 即时响应，不再只等 click 事件。
- 增大按钮点击面积，并加入 hover / pressed 即时反馈。
- 面板隐藏后释放鼠标捕获，避免桌宠窗口挡住桌面。

## V1.2.2 安装器场景可见性恢复

目标：用户打开 setup / installer 包时，桌宠在 UAC 或安装器窗口结束后能自动回到可见、置顶状态。

已落地策略：

- 开发启动脚本 `start.bat` 自动请求管理员权限。
- Windows 打包配置声明 `requestedExecutionLevel=requireAdministrator`，打包后的桌宠以管理员权限运行。
- 主窗口启动后定期做可见性自检。
- 如果桌宠被隐藏、最小化或压到普通窗口后面，自动 `showInactive`、重新置顶并 `moveTop`。
- 如果窗口跑到屏幕外，自动放回主屏幕工作区。
- 说明：Windows UAC 安全桌面期间普通应用无法显示在最上层，这是系统安全限制；确认 UAC 后会恢复。

## V2.0 语音交互版本

目标：新增麦克风语音交互，支持用户语音询问指引、语音确认下一步、操作提示语音播报。

计划能力：

- 用户说“我想下载 Steam，下一步怎么做”即可启动引导。
- 用户说“我完成了”“重新识别”“退出教程”控制流程。
- 桌宠用短语音播报当前操作提示。
- 保持文字气泡作为默认可视反馈，避免语音失败时流程中断。

## V3.0 摄像头多模态行为感知版本

目标：接入摄像头多模态行为感知，提高新手容错能力。

计划能力：

- 用户长时间找不到目标时，桌宠重复当前指引。
- 用户反复误操作时，桌宠放大提示或调整指向位置。
- 结合屏幕状态和用户行为状态，判断是否需要更细的解释。
- 摄像头能力默认关闭，必须由用户主动授权。
